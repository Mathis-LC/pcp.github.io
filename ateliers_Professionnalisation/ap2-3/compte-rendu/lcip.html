Après avoir installer sudo on va pouvoir mettre la vm en adresse IP statique

sudo nano /etc/network/interfaces

allow-hotplug enp3s0
                iface enp3s0 inet static 
                address 100.115.29.238/23 
                gateway 100.115.29.254 
                dns-nameservers 100.115.28.41 100.115.28.42

Puis on suit la documentation officielle pour installer Caddy

sudo apt install -y debian-keyring debian-archive-keyring apt-transport-https curl

curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | sudo gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg

curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | sudo tee /etc/apt/sources.list.d/caddy-stable.list

sudo chmod o+r /usr/share/keyrings/caddy-stable-archive-keyring.gpg

sudo chmod o+r /etc/apt/sources.list.d/caddy-stable.list

sudo apt update

sudo apt install caddy

Question

Caddy active-il par défaut l'affichage des dossiers?

Non caddy n'active pas l'affichage par défaut des dossiers

Quelle option permet de l'activer ?

sudo nano ./Caddyfile

:80 {
                        # Set this path to your site's directory.
                        root * /var/www/applifrais

                        # Enable the static file server.
                        file_server {
                            browse
                        }
                        # Another common task is to set up a reverse proxy:
                        # reverse_proxy localhost:8080

                        # Or serve a PHP site through php-fpm:
                        # php_fastcgi localhost:9000
                }

sudo systemctl reload caddy.service

L'option permettant d'activer l'affichage des dossiers est : 

        file_server {
                            browse
                        }

Comparer ce comportement avec celui d'apache et nginx en terme de sécurité.

Caddy est plus sécurisé avec sa configuration par défaut qu'Apache qui contrairement à Caddy affiche les dossiers. Cependant dans sa configuration par défaut Caddy à le même niveau de sécurité qu'Nginx puisque tous les deux n'affichent pas les dossiers par défaut.

Question

Essayez de naviguer sur le site, rencontrez-vous des erreurs 404?

Non, lorsque l’on navigue sur le site, on ne rencontre pas d’erreurs 404.

Analyser ce comportement sur la configuration par défaut de caddy?

La configuration par défaut de Caddy parvient à gérer à la fois les fichiers dynamiques et statiques, contrairement à Apache ou Nginx, pour lesquels il faut respectivement un fichier .htaccess ou modifier le fichier de configuration de Nginx.

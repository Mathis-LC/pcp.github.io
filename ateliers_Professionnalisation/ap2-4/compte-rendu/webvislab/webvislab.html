Configuration serveur web webvislab

choix des technologies pour le serveur webvislab

LAMP : Linux (US24), Apache2, Mysql, PHP

On importe le modèle de vm webvislab

On peut changer le nom de la machine

sudo nano /etc/hostname

Remplacer us2004 par webvislab

whereis apache2

Si non trouvé

sudo apt update

sudo apt install apache2 -y

Pour mettre en place le firewall

whereis ufw

Si non trouvé

sudo apt update

sudo apt install ufw -y

Configurer les règles du firewall

sudo ufw enable

sudo ufw app list

sudo ufw allow "Apache Full"

sudo ufw allow OpenSSH

sudo ufw status

Puis on créer les dossiers en suivant le tableau

sudo mkdir /var/www/Applifrais

sudo mkdir /var/www/Applifrais/2.0

sudo mkdir /var/www/Applifrais/beta

cd /etc/apache2/sites-available/

sudo nano applifraisv2.conf


                <VirtualHost *:80>
                    ServerName applifrais.webvislab.local

                    ServerAdmin webmaster@localhost
                    DocumentRoot /var/www/Applifrais/2.0/www
                    DirectoryIndex accueil.html index.html index.php

                    # Logs
                    ErrorLog ${APACHE_LOG_DIR}/error.log
                    CustomLog ${APACHE_LOG_DIR}/access.log combined

                    # Bonne pratiques de sécurité
                    <Directory /var/www/Applifrais/2.0/www>
                        # Désactive:
                        #   ExecCGI: l'exécution de scripts
                        #   Includes: l'inclusion de fichiers
                        #   Indexes: l'affichage du contenu des répertoires
                        Options -ExecCGI -Includes -Indexes

                        # Active les fichiers .htaccess
                        AllowOverride All 

                        # Autorise l'accès au contenu
                        Require all granted
                    </Directory>
                </VirtualHost>
